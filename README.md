# 학생 과제 디데이 웹앱

로그인, 서버, 데이터베이스 없이 브라우저에서 바로 실행되는 iPad용 과제 관리 웹앱입니다. `index.html`을 열면 과제를 추가하고, D-Day를 확인하고, 세부 체크리스트 진행률을 관리할 수 있습니다.

## 파일 구조

- `index.html`: 화면 구조와 입력 폼, 과제 목록 영역
- `style.css`: 모바일과 PC 반응형 디자인
- `app.js`: 과제 추가, 수정, 삭제, 정렬/필터, D-Day 계산, localStorage 저장과 복구
- `sample-data.js`: 처음 실행할 때 보여 줄 예제 과제 3개
- `tests.js`: 날짜 계산과 검증 로직 테스트
- `README.md`: 실행 방법, 데이터 구조, 테스트 결과

## Mac에서 실행하는 방법

1. Finder에서 이 폴더를 엽니다.
2. `outputs` 폴더 안의 `index.html`을 더블클릭합니다.
3. 브라우저가 열리면 바로 과제를 추가하거나 예제 과제를 수정해 볼 수 있습니다.

터미널로 열고 싶다면 아래 명령을 사용할 수 있습니다.

```bash
cd /Users/ahrahsehyuncho/Documents/Codex/2026-07-17/macos-index-html-30-1-2/outputs
open index.html
```

## 과제 데이터 구조

과제 하나는 아래와 같은 객체로 저장됩니다. 전체 과제 배열은 `localStorage`의 `student-dday-assignments-v1` 키에 JSON 문자열로 저장됩니다.

```js
{
  id: "assignment-고유값",
  title: "과제명",
  subject: "과목",
  dueDate: "2026-07-20",
  dueTime: "23:59", // 화면에는 표시하지 않지만 기존 데이터 호환과 기한 계산을 위해 유지
  priority: "보통", // 화면에는 표시하지 않지만 기존 데이터 호환을 위해 유지
  estimate: "1 ~ 10 사이의 시간 값",
  status: "시작 전 | 진행 중 | 작성 완료 | 제출 완료",
  memo: "메모",
  checklist: [
    { id: "check-고유값", text: "세부 단계", done: false }
  ],
  createdAt: "생성 시각 ISO 문자열",
  updatedAt: "수정 시각 ISO 문자열"
}
```

## 날짜와 시간 계산

- D-Day 계산은 `Asia/Seoul` 기준의 오늘 날짜를 사용합니다.
- 새 과제의 마감 시간은 화면에서 받지 않고 기본값 `23:59`로 저장합니다.
- 기존 데이터에 저장된 마감 시간은 오류 없이 읽어 기한 초과 계산에 사용합니다.
- D-Day 표시는 날짜 기준으로 하되, 저장된 마감 시간이 지나면 `기한 초과`로 표시합니다.
- 저장 데이터가 손상되거나 형식이 맞지 않으면 화면 상단에 경고를 보여주고 복구 가능한 데이터만 읽습니다.

## 백업과 복원

- `백업` 버튼을 누르면 현재 브라우저에 저장된 과제 목록을 JSON 파일로 내려받습니다.
- `복원` 버튼을 누르면 이전에 백업한 JSON 파일을 선택해 과제 목록을 다시 불러올 수 있습니다.
- 복원은 사용자가 확인했을 때만 현재 목록을 백업 파일 내용으로 대체합니다.
- 로그인이나 서버 저장 없이 같은 기기 또는 파일을 옮긴 다른 기기에서 데이터를 보관하기 위한 보조 기능입니다.

## iPad 홈 화면에 앱처럼 추가하기

기본 기능은 `index.html`을 더블 클릭해서 바로 사용할 수 있습니다. 다만 홈 화면 아이콘, 앱 이름, 기본 오프라인 캐시는 브라우저 보안 정책 때문에 `file://` 주소가 아니라 로컬 서버나 HTTPS 주소에서 더 안정적으로 동작합니다.

Mac에서 로컬 서버로 확인하려면 `outputs` 폴더에서 다음 명령을 실행합니다.

```bash
python3 -m http.server 8000
```

그다음 Safari에서 아래 주소를 엽니다.

```text
http://localhost:8000
```

iPad에서는 같은 네트워크에서 Mac의 로컬 주소로 접속하거나, 나중에 HTTPS로 배포한 주소를 Safari로 연 뒤 공유 버튼에서 `홈 화면에 추가`를 선택하면 됩니다. 이번 단계에서는 실제 앱스토어 앱, 로그인, 결제, 위젯 기능은 추가하지 않았습니다.

배포 준비와 iPad 확인 절차는 [iPad 홈 화면 추가와 HTTPS 배포 가이드](docs/ipad-pwa-deployment-guide.md)에 따로 정리했습니다.

## 테스트 실행 방법

브라우저 주소 끝에 `?runTests=1`을 붙여 열면 `tests.js`가 자동 실행됩니다.

```bash
open "index.html?runTests=1"
```

개발자 도구 Console에서 직접 실행할 수도 있습니다.

```js
runAssignmentTests()
```
